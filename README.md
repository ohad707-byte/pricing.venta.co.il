# Venta Auto Pricer v3 (OCR בלבד)

חילוץ ותמחור אוטומטי של פריטי **אוורור / פינוי עשן** מתוכניות PDF (מיוצאות
מ-AutoCAD), עם פלט Excel מפוצל לקטגוריות:

- **תעלות** (מלבניות / עגולות) - בנפרד מכל שאר הציוד
- **מפוחים ומשתיקים** - חובה: ספיקה (CFM), לחץ (Pa), קוטר/מידה, הספק (KW), רעש (dBA)
- **ונטות**, מפוצלות לפי יעד: **שירותים / מקלחת / מטבח** - כל אחת בטאב נפרד, עם אותם שדות חובה

**גרסה זו עצמאית לחלוטין**: חילוץ הנתונים מתבצע באמצעות Tesseract OCR
שרץ מקומית על המחשב. אין שום קריאה ל-API חיצוני, אין תלות ברשת בזמן
הריצה (מעבר להתקנה החד-פעמית של הספריות), ואין מפתחות API להזין בשום מקום.
מתאים במיוחד למחשבי עבודה ברשתות עם פילטור/פרוקסי אבטחה (כגון Cisco
Umbrella) שעלולים לחסום גישה לאפליקציות מאורחות בענן - הרצה מקומית
ב-`localhost` לא עוברת ברשת בכלל.

## מבנה הקוד

| קובץ | תפקיד |
|---|---|
| `categories.py` | הגדרת הקטגוריות, ה-regex לזיהוי, ומבנה הנתונים `ExtractedItem` |
| `extraction.py` | חילוץ פריטים מ-PDF באמצעות OCR (Tesseract + עברית) |
| `pricing.py` | ניהול מחירון: ברירות מחדל, טעינה מאקסל קיים, התאמת מחיר לפריט |
| `excel_export.py` | הפקת קובץ Excel עם טאב נפרד לכל קטגוריה + סיכום + לאימות |
| `app.py` | ממשק Streamlit שמחבר את כל המודולים |

## חשוב לדעת לפני שמריצים - שיטת קריאת התוכניות

תוכניות PDF המיוצאות מ-AutoCAD מכילות טקסט עברי **כתוויות מוטות על
קווי-חיבור (leader lines)**. נבדק בפועל: חילוץ טקסט סטנדרטי (`pdftotext`,
`PyMuPDF get_text("text")`) **מחמיץ כליל** את התוויות האלה ומחזיר בעיקר
שאריות גרפיות (GAS, X1, CFM) - לא את התיאורים המלאים שצריך בשביל כתב
כמויות אמיתי.

לכן הקוד **מרנדר כל עמוד לתמונה** ברזולוציה גבוהה, מחלק אותו לאריחים,
ומריץ Tesseract OCR (עם חבילת השפה העברית) על כל אריח בנפרד. זה עוקף את
מגבלת חילוץ-הטקסט הרגיל, אך התוצאה תלויה באיכות זיהוי-התווים של
Tesseract על טקסט מוטה/קטן/צפוף.

**לפני הסתמכות מלאה על התוצאה**: מומלץ לבדוק קובץ PDF אמיתי אחד, ולהשוות
ויזואלית (פתיחת התוכנית בצד תוצאות האפליקציה) אם הכמויות והערכים
(CFM/Pa/קוטר) שחזרו תואמים את מה שבאמת רשום בתוכנית. כל פריט בפלט עם
ציון "לאימות" או "חסר: ..." בעמודת ההערה מסמן בדיוק את הנקודות שדורשות
בדיקה אנושית.

## הרצה ב-Windows - שלב אחר שלב

1. **התקנת Python** (אם עדיין לא מותקן): הורד מ-**python.org** (גרסה 3.10
   ומעלה) - לא מ-Microsoft Store ולא מ-Chocolatey, כדי למנוע בעיות build.
   בזמן ההתקנה סמן **"Add python.exe to PATH"**.
   בדיקה ב-Command Prompt: `python --version`

2. **חילוץ הקבצים** לתיקייה, למשל `C:\venta_app\` (ודא שה-`.py` נמצאים ישירות
   בתיקייה, לא בתוך תיקייה מקוננת נוספת).

3. **התקנת התלויות**:
   ```
   cd C:\venta_app
   pip install -r requirements.txt
   ```
   כל החבילות ב-`requirements.txt` כוללות wheel מוכן ל-Windows ולא דורשות
   קומפילציה/Visual Studio.

4. **התקנת Tesseract + עברית** (שלב חובה - בלעדיו האפליקציה לא תוכל לחלץ
   נתונים): הורד והרץ את המתקין מ-
   https://github.com/UB-Mannheim/tesseract/wiki
   **בזמן ההתקנה חובה לסמן את חבילת השפה "Hebrew" ברשימת "Additional
   language data"** - זו לא ברירת מחדל מסומנת, צריך לחפש ולסמן אותה
   באופן יזום.

   אם לאחר ההתקנה האפליקציה עדיין לא מזהה את Tesseract, ייתכן שצריך
   להוסיף ידנית את `C:\Program Files\Tesseract-OCR` למשתנה הסביבה `PATH`
   (חיפוש "Environment Variables" בתפריט ההתחלה של Windows) ולפתוח
   Command Prompt חדש. (הקוד כבר בודק גם את מיקום ההתקנה הסטנדרטי הזה
   אוטומטית כגיבוי, ברוב המקרים לא יידרש שינוי ידני.)

5. **הרצה**:
   ```
   streamlit run app.py
   ```
   זה ייפתח טאב בדפדפן בכתובת `http://localhost:8501` - רץ רק על המחשב
   שלך, לא עובר ברשת.

## הרצה (Linux / macOS)

```bash
pip install -r requirements.txt
sudo apt-get install tesseract-ocr tesseract-ocr-heb   # Ubuntu/Debian
# או: brew install tesseract tesseract-lang             # macOS
streamlit run app.py
```

## פלט

קובץ Excel עם הגיליונות: `סיכום`, `תעלות`, `מפוחים ומשתיקים`,
`ונטות - שירותים`, `ונטות - מקלחת`, `ונטות - מטבח`, `אחר - לאימות`, `לאימות`
(האחרון מאגד כל פריט שביטחון הזיהוי שלו אינו "גבוה", או שחסר בו שדה חובה).

## מגבלות ידועות / נקודות לאימות ידני

- כמויות הנגזרות מ"מקדם קומות" (לפי שם קובץ כמו `קומה_2-6`) הן הערכה גסה -
  לא לוקחות בחשבון כללי חזרתיות מיוחדים (לדוגמה: פריט שחוזר רק בקומות זוגיות).
  יש לאמת ידנית כל פריט עם כמות שמוכפלת במקדם קומות גדול מ-1.
- שיוך ונטה ליעד (שירותים/מקלחת/מטבח) מתבסס על מילות מפתח בטקסט הסמוך - ברירת
  המחדל היא "שירותים" אם לא נמצאה מילת מפתח מובהקת. יש לבדוק את עמודת "הערה".
- כל פריט עם ציון "לאימות" או "חסר: ..." בעמודת ההערה דורש בדיקה אנושית מול
  התוכנית המקורית לפני סגירת תמחור.
- דיוק החילוץ תלוי באיכות ה-OCR על טקסט עברי מוטה - יש לבדוק ולכייל את
  הגדרות ה-DPI/חלוקת האריחים בסיידבר אם התוצאות חסרות פריטים שידוע
  שקיימים בתוכנית.
