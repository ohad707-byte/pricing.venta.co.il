# Venta Auto Pricer v3

חילוץ ותמחור אוטומטי של פריטי **אוורור / פינוי עשן** מתוכניות PDF (מיוצאות
מ-AutoCAD), עם פלט Excel מפוצל לקטגוריות:

- **תעלות** (מלבניות / עגולות) - בנפרד מכל שאר הציוד
- **מפוחים ומשתיקים** - חובה: ספיקה (CFM), לחץ (Pa), קוטר/מידה, הספק (KW), רעש (dBA)
- **ונטות**, מפוצלות לפי יעד: **שירותים / מקלחת / מטבח** - כל אחת בטאב נפרד, עם אותם שדות חובה

## מבנה הקוד

| קובץ | תפקיד |
|---|---|
| `categories.py` | הגדרת הקטגוריות, ה-regex לזיהוי, ומבנה הנתונים `ExtractedItem` |
| `extraction.py` | שני מסלולי חילוץ חלופיים (OCR / Claude API) - ראו למטה |
| `pricing.py` | ניהול מחירון: ברירות מחדל, טעינה מאקסל קיים, התאמת מחיר לפריט |
| `excel_export.py` | הפקת קובץ Excel עם טאב נפרד לכל קטגוריה + סיכום + לאימות |
| `app.py` | ממשק Streamlit שמחבר את כל המודולים |

## שני מסלולי חילוץ - חשוב לקרוא לפני שמריצים

תוכניות PDF המיוצאות מ-AutoCAD (כמו אלה שנבדקו בפרויקט הזה) מכילות טקסט עברי
**כתוויות מוטות על קווי-חיבור (leader lines)**. נבדק בפועל: חילוץ טקסט סטנדרטי
(`pdftotext`, `PyMuPDF get_text("text")`) **מחמיץ כליל** את התוויות האלה ומחזיר
בעיקר שאריות גרפיות (GAS, X1, CFM) - לא את התיאורים המלאים שצריך בשביל כתב כמויות
אמיתי. מכאן שני המסלולים:

### מסלול 1: OCR (`EXTRACTION_MODE=ocr`, ברירת מחדל)
מנדר כל עמוד לתמונה ברזולוציה גבוהה, מחלק לאריחים, ומריץ Tesseract עם חבילת
השפה העברית (`heb`) על כל אריח. **עצמאי לחלוטין** - לא תלוי ב-API חיצוני, לא
עולה כסף לפי בקשה. תלוי באיכות זיהוי-התווים על טקסט מוטה/קטן/צפוף - יש לבדוק
בפועל על תוכניות אמיתיות לפני הסתמכות מלאה על התוצאה.

דורש התקנת מערכת (לא רק pip):
```bash
# Ubuntu/Debian
sudo apt-get install tesseract-ocr tesseract-ocr-heb
# macOS
brew install tesseract tesseract-lang
```

### מסלול 2: Claude API (`EXTRACTION_MODE=claude`)
שולח כל אריח תמונה ל-Anthropic API עם פרומפט שמבקש JSON מובנה של הפריטים
שזוהו. מדויק יותר על תוויות מוטות (נבדק ידנית בפרויקט המקור והוכח שעובד), אך
עולה כסף לפי קריאה ודורש רשת + מפתח API (`ANTHROPIC_API_KEY`).

### החלטה בין המסלולים
האפליקציה מאפשרת לבחור בסיידבר בכל הרצה. מומלץ: **לבדוק קודם את מסלול ה-OCR
על קובץ PDF אמיתי אחד**, ולהשוות ויזואלית (פתיחת התוכנית + עיון ידני) אם הכמויות
והערכים (CFM/Pa/קוטר) שחזרו תואמים את מה שבאמת רשום בתוכנית. אם לא - לעבור
למסלול Claude API.

## הרצה

```bash
pip install -r requirements.txt
# מסלול OCR בלבד דורש גם:
#   sudo apt-get install tesseract-ocr tesseract-ocr-heb
streamlit run app.py
```

## פלט

קובץ Excel עם הגיליונות: `סיכום`, `תעלות`, `מפוחים ומשתיקים`,
`ונטות - שירותים`, `ונטות - מקלחת`, `ונטות - מטבח`, `אחר - לאימות`, `לאימות`
(האחרון מאגד כל פריט שביטחון הזיהוי שלו אינו "גבוה", או שחסר בו שדה חובה).

## מגבלות ידועות / נקודות לאימות ידני

- כמויות הנגזרות מ"מקדם קומות" (לפי שם קובץ כמו `קומה_2-6`) הן הערכה גסה -
  לא לוקחות בחשבון כללי חזרתיות מיוחדים (לדוגמה: פריט שחוזר רק בקומות זוגיות).
  יש לאמת ידנית כל פריט עם כמות שמוכפלת במקדם קומות גדול מ-1.
- שיוך ונטה ליעד (שירותים/מקלחת/מטבח) מתבסס על מילות מפתח בטקסט הסמוך - ברירת
  המחדל היא "שירותים" אם לא נמצאה מילת מפתח מובהקת. יש לבדוק את עמודת "הערה".
- כל פריט עם ציון "לאימות" או "חסר: ..." בעמודת ההערה דורש בדיקה אנושית מול
  התוכנית המקורית לפני סגירת תמחור.
